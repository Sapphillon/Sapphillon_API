## Research Results: Google API Proto Style — Documentation & Commenting Conventions

### Summary
Google AIP-192ではprotoファイル内の**全公開コンポーネント**（service, rpc, message, field, enum, enum value）に**leading `//` commentが必須**。コメントはCommonMark形式、最初の文は主語省略の三人称現在形（例: `// Creates a book.`）。buf.ioの`COMMENTS`カテゴリが7つの個別ルール（`COMMENT_SERVICE`, `COMMENT_RPC`, `COMMENT_MESSAGE`, `COMMENT_FIELD`, `COMMENT_ENUM`, `COMMENT_ENUM_VALUE`, `COMMENT_ONEOF`）で有無を自動検査可能。現在のコードベースは概ね良いコメント率（平均〜53%）だが、AIP-192で禁止の構造化セクション（`// Fields:`, `// Example:` + コードブロック）を使用しており、文体も一部不一致。

### Key Findings
1. **必須要素**: AIP-192はservice, rpc, message, field, enum, enum valueのすべてにleading comment必須。buf lintの`COMMENTS`カテゴリで機械検査可能
2. **コメント形式**: `//` のみ使用、`/* */` 禁止。Leading commentsのみがツールで認識される（trailing/detachedは無視）
3. **文体**: 主語省略の三人称現在形、大文字開始・ピリオド終了。見出し/テーブル/HTML/ASCII Art 禁止
4. **非推奨**: `option deprecated = true;` + 最初の行 `// Deprecated: <reason>`
5. **既存コードベースの課題**: `// Fields:` / `// Example:` + コードブロック(````) はAIP-192非推奨。ai/v1/ パッケージのコメント率が低い（20-48%）

### Recommended Approach
- `buf.yaml`に`COMMENTS`カテゴリを追加してlint自動化
- 既存の`// Fields:` / `// Example:` 構造化セクションをAIP-192準拠のフリーテキストに移行
- コメント文体を「主語省略の三人称現在形」に統一
